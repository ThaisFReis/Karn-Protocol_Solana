## Critérios de Aceitação

### KRN-02 — Snapshot Voting

Mana é calculada em `proposal.creation_time`, não em `Clock::get()` no momento do voto.

**Implementação:** `cast_vote::handler` passa `creation_time` (não `now`) para `calculate_mana`:

```rust
let voting_power = calculate_mana(
    stats.credential_level,
    stats.permanent_level,
    stats.credential_expiry,
    stats.activity_level,
    stats.activity_expiry,
    creation_time,  // KRN-02: snapshot, not now
);
```

**Teste:** `tests/krn/krn-02.spec.ts`  
Voter com `credential_expiry = EPOCH_T + VOTING_DELAY + 1`. Vota 100s depois da expiração.  
`for_votes = 6` (snapshot) ≠ `5` (corrente após expirar).

### KRN-03 — Participation Threshold

Participação `< 4%` de `total_mana_at_creation` → `Defeated`, independente de `for_pct`.

**Implementação:** `proposal_state()` checa participation ANTES de quorum:

```rust
let participation = (total_votes as u128).saturating_mul(100) / total_mana;
if participation < 4 { return ProposalState::Defeated; }
```

**Teste:** `tests/krn/krn-03.spec.ts`  
`total_supply = 1000` → `total_mana_at_creation = 5000`. Voter mana = 6 (< 200 = 4% de 5000).  
100% votos FOR → `Defeated` (participação = 0% < 4%).

### DT-05 — Governor Config PDA como signer de CPIs

```rust
let seeds: &[&[u8]] = &[GOVERNOR_CONFIG, &[bump]];
let signer = &[seeds];
```

Cada variante de `ProposalAction` usa `CpiContext::new_with_signer` com esse signer.

### DT-06 — Reentrancy Guard

`GovernorConfigPda.locked` é setado `true` antes de qualquer CPI e restaurado `false` ao final.  
`require!(!ctx.accounts.config.locked, GovernorError::ReentrancyDetected)` no início dos handlers.

### Anti-double-vote

`Vote` PDA com `seeds=[VOTE, proposal_id_le8, voter_pubkey]`. Existência = tentativa de double-vote → `AccountAlreadyInUse` do runtime Anchor.

## Decisões Técnicas

### Borrow conflict em execute.rs

A ação precisa ser clonada e o `bump` capturado antes de qualquer borrow mutável em `ctx.accounts.config`:

```rust
let action = ctx.accounts.proposal.action.clone();
let bump = ctx.accounts.config.bump;
let gov_pda = ctx.accounts.config.to_account_info();
ctx.accounts.config.locked = true;
ctx.accounts.proposal.executed = true;
```

### remaining_accounts por variante

Cada `ProposalAction` usa `remaining_accounts` com layout fixo (documentado em `execute.rs`). O `execute` não tem contas adicionais no `Accounts` struct — tudo vem via `remaining_accounts`.

### UpdateGovernanceConfig sem CPI

A única variante que não usa CPI. Atualiza `params` PDA diretamente (`ctx.accounts.params`).

### Bankrun: injeção sintética do Valocracy Config

O teste 10 (ValocracyPauseCredit CPI) exige que `valocracy_config.governor == govConfigPda`. Caso contrário, `has_one = governor` falha na validação da CPI. A função `injectValocracyConfig` em todos os testes do Governor injeta o `govConfigPda` correto.
