### `UsedNonce` PDA (state.rs)

```rust
#[account]
pub struct UsedNonce { pub bump: u8 }
```

A existência da PDA derivada de `[USED_NONCE, caller, nonce]` **é** o marcador de consumo. O Anchor `init` constraint falha automaticamente se a PDA já existir → replay protection sem código adicional.

### `self_register` instruction

`programs/valocracy/src/instructions/self_register.rs`. Fluxo:

1. **Verifica `expiry > Clock::unix_timestamp`** (`SignatureExpired`)
2. **Walk Instructions sysvar** procurando uma chamada ao `Ed25519SigVerify` precompile cujo:
   - `program_id == Ed25519SigVerify111111111111111111111111111`
   - data tem `num_signatures == 1`
   - pubkey embedded == `Config.signer`
   - message embedded == `build_self_register_payload(caller, nonce, expiry, track_id)`
3. Se nenhuma instrução match → `InvalidSignature`
4. Cria `UsedNonce(caller, nonce)` PDA via `init` (falha se replay)
5. Set `caller_stats.primary_track_id = Some(track_id)` (D3-A)
6. Reusa `apply_mint` (de M4) para mintar member badge: incrementa `total_supply`, cria `TokenOwner`/`TokenValorId`, atualiza `credential_level`/`credential_expiry`, emite `MintEvent`

### Decisões de implementação importantes

- **Ed25519 verificação por content-match, não por índice.** Cliente não passa `sig_ix_index` — o programa varre o sysvar procurando uma instrução do precompile que assinou exatamente o payload esperado contra `Config.signer`. Isso impede ataques onde o cliente aponta para um precompile genuíno (mas não relacionado) na mesma tx para "passar" pela validação.

- **Sig math offload.** O programa não recomputa Ed25519 — apenas confirma que o precompile RAN com o input correto. Se o precompile rodou e a tx não falhou, a assinatura é matematicamente válida. Isso economiza ~20-30k CU por tx.

- **D3-A:** O payload Solana tem `track_id` ao final (Stellar não tem). Documentado em CONFIG.md DT-10. Permite setar `primary_track_id` no register sem follow-up `update_primary` proposta.

- **`Ed25519SigVerify` ID hardcoded** via `anchor_lang::pubkey!("Ed25519SigVerify111111111111111111111111111")` — `solana_program::ed25519_program` não é re-exportado de forma estável por `anchor-lang` 0.32.

## Erros encontrados

- **`anchor_lang::solana_program::ed25519_program::ID` não existe.** anchor-lang 0.32 não re-exporta esse path. Fix: hardcode via `anchor_lang::pubkey!(...)` macro.
- **clippy `needless_loop`.** A primeira versão do walk usava `loop { match ... }`. Refator para `while let Ok(ix) = ...` (clippy clean + idiomático).
- **`@types/tweetnacl` não existe no npm.** `tweetnacl` ships own `nacl.d.ts`. Removido `@types/tweetnacl` do install.

Nenhum dos três custou >10 minutos. Não justifica entrada em `errors/`.

## Caveats / dívidas técnicas

- **Backend signing service não implementado.** O programa valida assinaturas mas o **endpoint backend que assina** payloads para usuários está fora do escopo do contrato. Para o demo do hackathon, scripts/CLI vão simular o backend gerando keypair offline e assinando localmente.
- **Treasury CPI ainda não plugado.** `apply_mint` (chamado por `self_register`) ainda não faz CPI para `Treasury.deposit`. Aguarda M11. Member com `member_rarity = 5` shares conceituais mas não materializadas ainda.
- **Member badge não pode ser revogado.** `revoke` (M9) suporta isso — mas se um membro foi self-registered e depois revoke acontece, o `UsedNonce` PDA permanece, então re-registro com mesmo nonce continua bloqueado. Comportamento correto (nonce é assinatura única do backend; backend deve emitir novo nonce em re-onboarding).
- **`Config.signer` não pode ser rotacionado.** Rotação vira `update_signer(new_signer)` instrução Governor-only — adiar para módulo apropriado se ficar relevante. Não bloqueia hackathon.
