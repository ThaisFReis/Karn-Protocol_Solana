## Escopo entregue

### Instrução — `programs/valocracy/src/instructions/update_primary.rs`

`update_primary(account, new_track_id, new_valor_id)` — Governor-only.

**Fluxo:**
1. `has_one = governor` no `Config` PDA verifica autoridade (`NotAuthorized` se falhar)
2. Carrega `UserStats(account)` via seeds `[USER_STATS, account]`
3. `stats.primary_track_id = Some(new_track_id)`; `stats.primary_valor_id = Some(new_valor_id)`
4. Emite `PrimaryUpdatedEvent { account, new_track_id, new_valor_id }`

**Efeito nos mints subsequentes:** `effective_rarity` já usava `primary_track_id` desde M4. Após `update_primary`, badges com `track_id == new_track_id` creditam `rarity` (plena); outros creditam `secondary_rarity`.

### Wiring

- `instructions/mod.rs`: `pub mod update_primary` + `pub use update_primary::*`
- `lib.rs`: entrada `update_primary` no `#[program]`

## Critério de aceite (do PRD)

- [x] `update_primary` chamado por wallet arbitrária retorna `NotAuthorized`
- [x] Após update, `effective_rarity_for(account, valor)` reflete novo track
