**Data:** 2026-05-01  
**Status:** ✅ Concluído

## Objetivo

Implementar o programa `treasury` no Solana com:
- Inicialização do `TreasuryState` PDA + vault ATA
- Instrução `transfer` Governor-only com reentrancy guard
- Vault math puro em `karn-shared` (paridade KRN-01)

---

## Arquivos Criados / Modificados

### `crates/karn-shared/src/vault.rs` (novo)

Pure functions para vault math ERC4626-style com virtual offsets:

```rust
pub const VIRTUAL_SHARES: u128 = 1_000;
pub const VIRTUAL_ASSETS: u128 = 1;

pub fn total_assets(vault_balance: u64, restricted_reserves: u64) -> u64
pub fn convert_to_assets(shares: u128, total_shares: u128, total_assets_val: u64) -> u64
pub fn convert_to_shares(assets: u64, total_shares: u128, total_assets_val: u64) -> u128
```

Intermediários u128 para evitar overflow. Os offsets virtuais previnem first-depositor inflation attack.

### `programs/treasury/src/state.rs` (novo)

```rust
pub struct TreasuryState {
    pub governor: Pubkey,           // 32
    pub valocracy: Pubkey,          // 32
    pub asset_mint: Pubkey,         // 32
    pub total_shares: u128,         // 16
    pub restricted_reserves: u64,   //  8
    pub locked: bool,               //  1
    pub bump: u8,                   //  1
}
// SIZE = 122 bytes (+ 8 discriminador = 130 no account)
```

### `programs/treasury/src/instructions/initialize.rs`

- Seeds PDA: `[TREASURY_STATE]` = `[b"treasury"]`
- Vault ATA: `associated_token::mint = asset_mint, authority = state`
- Inicializa: governor, valocracy, asset_mint, total_shares=0, restricted_reserves=0, locked=false, bump

### `programs/treasury/src/instructions/transfer.rs`

Guards (nesta ordem):
1. `amount > 0` → `ZeroAmount`
2. `!state.locked` → `ReentrancyDetected`
3. `vault_ata.amount >= amount` → `InsufficientAssets`

Fluxo:
```
state.locked = true
  → token::transfer CPI com PDA signer [TREASURY_STATE, &[bump]]
state.locked = false
emit!(Transfer { receiver, amount })
```

---

## Decisões de Design

**D1 — Reentrancy guard via `locked` bool:** Anchor faz rollback automático se o CPI falhar, então `locked` nunca fica permanentemente travado. O guard é defensivo contra CPIs recursivas externas.

**D2 — `receiver_ata: AccountInfo` (não `Account<TokenAccount>`):** Valida mint+owner implicitamente via SPL transfer. Evita constraint `associated_token` que exigiria passar `mint` como conta extra.

**D3 — KRN-01 via `restricted_reserves`:** `total_assets = vault_balance - restricted_reserves`. Quando Labs estiver implementado (M12), fundos de scholarship incrementam `restricted_reserves`, excluindo-os do cálculo de shares.
