## Erros encontrados

Nenhum erro não-trivial. Dois caveats de implementação resolvidos durante o módulo:

1. **`as u32` em variant não retorna o offset Anchor.** Tentei testar `assert_eq!(ValocracyError::AlreadyInitialized as u32, 6000)` → arithmetic overflow porque `as u32` retorna o **discriminante raw** (0-indexed), não o código exposto ao cliente. Anchor adiciona +6000 só na conversão `Into<Error>`. Fix: testar discriminante direto (`as u32 == 0`) e documentar o mapping em ADR-0002.
2. **Clippy `assertions_on_constants`.** `assert!(CONST <= 100)` em `#[test]` é tautológico em runtime. Fix: mover pra `const _: () = assert!(...)` (compile-time). Mesma proteção, sem lint.

## Caveats / dívidas técnicas

- **Eventos não têm topics.** Stellar `#[topic]` distingue campos indexados; Anchor não tem o conceito (logs são parseados por field name). Indexer/SDK off-chain (M15+) tratará isso.
- **Account structs ainda não existem.** A tabela `docs/PDA_CONVENTIONS.md` lista o conteúdo esperado das PDAs; as `#[account]` structs reais virão em M3 (Valocracy state), M10 (Treasury state), M13 (Governor state).
- **Module re-exports preguiçosos.** Cada programa hoje tem `pub use errors::ValocracyError` mas não re-exporta os events. Quando M3+ começar a usar eventos, eu adiciono `pub use events::*` se for útil.
