## Escopo entregue

### `karn-shared::mana` (módulo novo)

`crates/karn-shared/src/mana.rs`:

- `calculate_mana(credential_level, permanent_level, credential_expiry, activity_level, activity_expiry, current_time) -> u64` — função pura, sem I/O
- Aritmética `u128` interna em todos os intermediários (KRN-04) — sem risco de overflow mesmo com `credential_level = u64::MAX`
- Paridade com a implementação Stellar/Soroban: mesma fórmula de três componentes (floor + credential_bonus + activity_bonus)
- `MEMBER_FLOOR` (5) garantido como piso mesmo com todos os bônus zerados

**Fórmula implementada (DT-10 D2):**

```
credential_bonus =
  se now < credential_expiry:
    (credential_level − MEMBER_FLOOR) × (expiry − now) / VACANCY_PERIOD
  senão se permanent_level > 0:
    permanent_level   (fallback Stellar; sempre 0 em v2-Solana)
  senão:
    0

activity_bonus =
  se activity_level > 0 && now < activity_expiry:
    activity_level × (expiry − now) / ACTIVITY_PERIOD
  senão:
    0

Mana = MEMBER_FLOOR + credential_bonus + activity_bonus
```

  - Chamar via `.view()` — não muta estado
- **`get_votes_at(account, timestamp)`** — `programs/valocracy/src/instructions/get_votes_at.rs`
  - Snapshot histórico para KRN-02 (voting power na criação da proposta)

Ambas as instruções usam `UncheckedAccount` com verificação de seeds para suportar consulta de conta não inicializada sem erro de desserialização.
