# M11 — Allocation de Shares (CPI Valocracy → Treasury)

**Data:** 2026-05-01  
**Status:** ✅ Concluído  
**Testes:** 4/4 passando (54 total no suite)

---

## Objetivo

Permitir que Valocracy aloque shares em mints, mantendo o vínculo contribuição → ownership econômica.

## Arquivos Criados / Modificados

### `programs/treasury/src/state.rs` (atualizado)

Adicionada `UserShares` struct:
```rust
#[account]
pub struct UserShares {
    pub owner: Pubkey,   // 32
    pub shares: u128,    // 16

impl UserShares { pub const SIZE: usize = 49; }
```
Seeds: `[USER_SHARES, owner.as_ref()]` = `[b"shares", owner]`.

### `programs/treasury/src/instructions/deposit.rs` (novo)

- `valocracy_authority: Signer` — deve ser o Config PDA do Valocracy
- Constraint: `valocracy_authority.key() == state.valocracy @ NotAuthorized`
- Chamada direta por wallet qualquer retorna `NotAuthorized`
- Primeiro depósito (`total_shares == 0`): `shares >= MIN_INITIAL_DEPOSIT (1000)` ou `InsufficientShares`
- `init_if_needed` no `UserShares` PDA (payer = minter forwarded via CPI)
- Incrementa `user_shares.shares` e `state.total_shares` com `saturating_add`
- Emite `Deposit { receiver, shares }`

### `programs/valocracy/src/instructions/mint.rs` (atualizado)

Mudanças:
1. `apply_mint` retorna `Result<u64>` (effective rarity) em vez de `Result<()`
2. Handler usa `remaining_accounts` para CPI opcional:
   - `remaining_accounts[0]` = treasury program
   - `remaining_accounts[1]` = treasury state PDA (writable)
   - `remaining_accounts[2]` = user_shares PDA (writable, init_if_needed)
3. CPI sign via seeds `[VALOCRACY_CONFIG, &[config_bump]]` (Config PDA signer)
4. Sem remaining_accounts → CPI skip (garante retrocompatibilidade com 44 testes M4-M9)

**Lifetime fix:** `pub fn mint<'info>(ctx: Context<'_, '_, '_, 'info, Mint<'info>>)` — necessário para usar `config.to_account_info()` após o borrow mutável em `apply_mint`.

### `programs/valocracy/src/instructions/guardian_mint.rs` e `self_register.rs`

Atualizados: `apply_mint(...)?;` → `apply_mint(...)?; Ok(())` para acomodar o novo tipo de retorno `Result<u64>`.

### `programs/treasury/Cargo.toml`

## Decisões de Design

**D1 — remaining_accounts para CPI opcional:** Mantém a assinatura da instrução `mint` idêntica para os testes M4-M9 (sem remaining_accounts, CPI é skipped). Testes M11 passam 3 remaining_accounts extras.

**D2 — Config PDA como valocracy_authority:** `state.valocracy` guarda o endereço do Config PDA do Valocracy. A CPI assina com seeds `[b"config", &[bump]]`, produzindo exatamente esse PDA. Chamadas diretas por wallets comuns falham no constraint de autoridade.

**D3 — apply_mint retorna effective rarity:** Necessário para passar o valor correto de shares para o deposit. Os outros callers (`guardian_mint`, `self_register`) ignoram o valor de retorno com `?; Ok(())`.

**D4 — MIN_INITIAL_DEPOSIT = 1000:** Âncora da exchange rate inicial contra ataques de inflação first-depositor (ERC-4626 §5). Rarity < 1000 em primeiro mint causa `InsufficientShares` propagado de volta via CPI error.
