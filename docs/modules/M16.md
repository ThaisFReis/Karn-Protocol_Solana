**Data:** 2026-05-06  
**Status:** Concluído

## Escopo entregue

- `sdk/src/react/` criado como camada React opcional sobre o SDK
- `KarnSolanaProvider` com:
  - autodetecção de cluster via env
  - resolução de RPC por cluster
  - injeção de `Connection`
  - suporte a `wallet`, `provider`, `programs` ou `clients`
- `createKarnClients(programs)` para montar `ValocracyClient`, `GovernorClient` e `TreasuryClient`
- `createAnchorProvider(connection, wallet)` para conectar wallets compatíveis com a interface esperada
- hooks:
  - `useKarnSolana`
  - `useValocracy`
  - `useGovernor`
  - `useTreasury`
- export subpath `@karn_lat/protocol-sdk-solana/react`
- `peerDependency` de `react`
- exemplos CSF em `sdk/examples/` para cada hook

## Comportamento dos hooks

### `useValocracy`

- lê `UserStats` do wallet atual
- calcula `mana`
- expõe `refresh`
- expõe helpers de transação:
  - `register`
  - `mint`
  - `mintCommunity`
  - `guardianMint`
  - `creditActivity`
  - `revoke`
  - `setVerified`
  - `pauseCredit`
  - `resumeCredit`

### `useGovernor`

- lê `GovernorConfigPda` e `GovernanceConfig`
- expõe `refresh`
- expõe:
  - `getProposal`
  - `computeProposalState`
  - `propose`
  - `vote`
  - `execute`

### `useTreasury`

- lê `TreasuryState`, `UserShares` e `Claimable`
- expõe `refresh`
- expõe:
  - `getLab`
  - `totalAssets`
  - `fundLab`
  - `withdrawScholarship`

## Limites conhecidos

- O repo não inclui um app React real nem runtime de Storybook; os exemplos em `sdk/examples/` são templates de integração.
- Como o monorepo não carrega dependências de wallet adapter React neste checkout, a integração final com Phantom/Backpack/Solflare depende do app consumidor fornecer a wallet concreta.
