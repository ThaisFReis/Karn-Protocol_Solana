## Decisões

### Por que não Codama?

O PRD cita Codama como tooling de geração de clientes. Após análise:
- Codama não está no projeto e exigiria configuração adicional + geração de código
- Os IDLs já têm tipos TypeScript gerados pelo Anchor em `target/types/*.ts`
- Os testes existentes (M1–M14) já usam `@coral-xyz/anchor` com esses tipos
- Para hackathon ("mínimo viável"), thin wrappers sobre o Program Anchor são mais coesos com o resto do repo

Codama pode ser adicionado em M17/M18 se necessário para o demo dApp.

### BN → BigInt conversão

O Anchor usa `BN.js` para u64/i64 nas contas fetchadas. Os clients fazem `v.toString()` antes de `BigInt()` para não depender de BN ser assignable a bigint — esse padrão é explícito para não quebrar em futuras versões do Anchor.

### ProposalAction.ValocracyRevoke — extraAccounts

`buildExecuteRemainingAccounts` para `ValocracyRevoke` requer o owner do token via `extraAccounts.receiverAta`. O campo foi reutilizado por conveniência de API (evita novo parâmetro). Documentado na JSDoc do helper.
