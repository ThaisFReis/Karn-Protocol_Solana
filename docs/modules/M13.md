### `programs/governor/src/instructions/propose.rs` (novo)

Handler de criação de proposta com leitura cross-program (DT-04):

```rust
// Contas lidas diretamente da Valocracy via seeds::program (sem CPI)
proposer_stats: Account<'info, valocracy::state::UserStats>,
    seeds = [USER_STATS, proposer.key().as_ref()],
    seeds::program = config.valocracy,

valocracy_config: Account<'info, valocracy::state::Config>,
    seeds = [VALOCRACY_CONFIG],
    seeds::program = config.valocracy,
```

Fluxo do handler:
1. Valida `description.len() <= 500`
2. Calcula `mana` com `calculate_mana()` de `karn-shared`
3. Verifica `mana >= params.proposal_threshold` (erro `NoVotingPower`)
4. **KRN-02 snapshot:** `total_mana_at_creation = total_supply × MEMBER_FLOOR`
5. Calcula `start_time = now + voting_delay`, `end_time = start_time + voting_period`
6. Inicializa todos os campos do `Proposal`
7. Incrementa `config.proposal_count` com `saturating_add`
8. Emite `ProposalCreated { proposal_id, proposer }`

## Decisões Técnicas

| Decisão | Motivo |
|---------|--------|
| `seeds::program = config.valocracy` para ler contas | DT-04: sem CPI, leitura direta cross-program |
| `valocracy = { features = ["no-entrypoint"] }` no Cargo.toml | Importa tipos sem duplicar entry point |
| Re-export glob em `instructions/mod.rs` | Necessário para `#[program]` macro resolver `__client_accounts_*` |
| `total_mana_at_creation = total_supply × MEMBER_FLOOR` | KRN-02: snapshot conservador que garante que abstenções não distorcem o quórum |
| Injeção de contas sintéticas nos testes | Isola lógica do governor sem depender do flow Ed25519 da Valocracy |
