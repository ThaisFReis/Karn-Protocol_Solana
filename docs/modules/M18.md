**Data:** 2026-05-06  
**Status:** Concluído

## O que entrou

- `migrations/deploy.ts` saiu de stub para bootstrap operacional de devnet
- `deployments/devnet.json` passou a registrar snapshot completo do ambiente
- `scripts/demo.ts` executa o fluxo de demo a partir do snapshot
- `valocracy.update_governor`
- `valocracy.update_treasury`
- `treasury.update_governor`
- novas `ProposalAction` para rotação futura de autoridade via Governor:
  - `TreasuryUpdateGovernor`
  - `ValocracyUpdateGovernor`
  - `ValocracyUpdateTreasury`

## Fluxo final do bootstrap

1. Inicializa `valocracy`, `governor` e `treasury`
2. Faz seed de `Valor`, `Guardian`, `CreditAuthority`, badge de demo e `Lab`
3. Atualiza `valocracy.treasury` para a `TreasuryState` PDA
4. Rotaciona `valocracy.governor` para `gov_config`
5. Rotaciona `treasury.governor` para `gov_config`
6. Escreve snapshot final com `authorityMode = "governance-pda"`

## Resultado operacional

- a wallet humana continua existindo apenas como bootstrap signer
- depois do seed, os programas deixam de depender dela como autoridade persistente
- o estado salvo no snapshot já representa a topologia final de governança on-chain
