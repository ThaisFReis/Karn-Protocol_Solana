## Escopo entregue

### State

Adicionado em `programs/valocracy/src/state.rs`:

- **`GuardianTracks`** — authority Pubkey, `Vec<u64>` track_ids (cap `MAX_TRACKS = 32`, ver DT-08), bump
- Errors expandidos com 2 variantes Solana-only (após o bloco de paridade Stellar):
  - `InvalidTokenId` (índice 18) — token_id mismatch quando cliente passa valor != `total_supply + 1`
  - `GuardianSelfMintForbidden` (índice 19) — KRN-05 reforçado: `guardian == account` rejeitado mesmo com dual `Signer`

### Helpers (`programs/valocracy/src/helpers.rs`)

- **`BadgeCategory`** enum — Member, Founder, Leadership, Track, Community, Governance
- **`get_badge_category(valor_id) -> Result<BadgeCategory>`** — mapa por range (DT-09)
- **`effective_rarity(primary_track_id, &Valor) -> u64`** — Stellar parity:
  - `track_id == 0` → `rarity` (domain-agnostic)
  - `primary_track_id == valor.track_id` → `rarity`
  - else → `secondary_rarity`
- **`effective_rarity_for(&UserStats, &Valor)`** — wrapper conveniente

### Instructions

- **`mint(valor_id, token_id)`** — `programs/valocracy/src/instructions/mint.rs`. Governor-only para Leadership/Track/Governance. Member/Founder rejeitam com `BadgeNotMintable`; Community rejeita com `MintNotAuthorized` (rota deferida — ADR-0004). Helper `apply_mint` extraído como `pub(crate)` para reuso.
- **`guardian_mint(valor_id, token_id)`** — `programs/valocracy/src/instructions/guardian_mint.rs`. Dual-auth (`guardian` + `account` ambos `Signer`). KRN-05 + CONFIG.md Rule 5 reforçados via `require_keys_neq!`. Verifica:
  1. `guardian != account`
  2. `valor.track_id != 0` (só badges domain-bound)
  3. `GuardianTracks(guardian).track_ids` contém `valor.track_id`
  4. Categoria é `Track` (defensivo)
- **`set_guardian_tracks(guardian_key, track_ids)`** — `programs/valocracy/src/instructions/set_guardian_tracks.rs`. Governor-only. `init_if_needed` para create+update. Valida cap de 32 tracks.
- **`remove_guardian(guardian_key)`** — `programs/valocracy/src/instructions/remove_guardian.rs`. Governor-only. `close = governor` retorna rent.

## Caveats / dívidas técnicas

- **`mint_community` deferred** (ADR-0004). Community badges (60–69) não são minteáveis até essa instrução chegar. Não bloqueia demo.
- **Treasury CPI no apply_mint não implementado.** Stellar's `mint_internal` faz `treasury.deposit(account, effective_rarity)` no fim. Solana version não — aguarda M11 (Valocracy → Treasury CPI). Comportamento de UserStats está completo; apenas a integração econômica falta.
- **`Config.credit_paused` não foi usado em M4.** Vai ser exercido em M7 (`credit_activity`).
- **`UserStats.activity_*` campos zerados em mint** — esperado, são v2 features que entram em M7.
- **Solana-only error variants em índices 18/19.** Documentado em `errors.rs` header e em ADR-0002 (paridade Stellar mantém-se via "primeiros 18 índices == Stellar 1..18"). Variant count test agora trava em 20.
