## Escopo entregue

### State (`programs/valocracy/src/state.rs`)

5 structs com `#[account]`:

- **`Config`** (singleton) — governor, treasury, signer (32 bytes), member_valor_id, leadership_valor_id, total_supply, credit_paused, bump
- **`Valor`** — rarity, secondary_rarity, track_id, metadata (≤200 bytes), bump
- **`UserStats`** — credential_level, permanent_level (DT-10 D1: sempre 0 em v2-Solana), credential_expiry, verified, primary_track_id (Option), primary_valor_id (Option), activity_level, activity_expiry, bump
- **`TokenOwner`** — owner Pubkey, bump
- **`TokenValorId`** — valor_id u64, bump

Cada struct tem `pub const SIZE: usize` calculada para uso com `space = 8 + Self::SIZE` em `#[account(init)]`.

### Instructions

- **`initialize(governor, treasury, signer: [u8;32], member_valor_id, leadership_valor_id)`** — `programs/valocracy/src/instructions/initialize.rs`. Cria Config PDA singleton. Emite `InitializedEvent { genesis_count: 0 }`.
- **`set_valor(valor_id, rarity, secondary_rarity, track_id, metadata)`** — `programs/valocracy/src/instructions/set_valor.rs`. Governor-only via `has_one`. Cria/atualiza Valor PDA com `init_if_needed`. Emite `ValorUpdateEvent`.

### Wiring

- `lib.rs` declara `pub mod state; pub mod instructions;` e expõe as 2 instruções via `#[program]`
- `instructions/mod.rs` glob-exporta as Accounts structs (com `#[allow(ambiguous_glob_reexports)]` justificado)
- Cargo.toml: `anchor-lang` agora com feature `init-if-needed` ativada

## Erros encontrados

- [Ambiguous glob re-export entre `initialize::handler` e `set_valor::handler`](../errors/ambiguous-glob-reexport.md) — ambos `instructions::*` re-exportavam um `handler` cada; clippy `-D warnings` reprovou. Fix: `#[allow(ambiguous_glob_reexports)]` localizado em `instructions/mod.rs` (a ambiguidade é benigna pq todas chamadas qualificam o path).
- **anchor-bankrun 0.5 vs Anchor 0.32 peer-dep mismatch** — `npm install` reclamou que anchor-bankrun expects `@coral-xyz/anchor@^0.30.0`. Fix: `--legacy-peer-deps`. Anchor 0.32 IDL parser dentro do Program constructor lida com a IDL corretamente em runtime.
- **`anchor build -- --features init-if-needed` falhou** — `--features` propaga pra todos os workspace members; `governor` e `treasury` não declaram a feature. Fix: passar `init-if-needed` como feature default do `anchor-lang` no Cargo.toml de valocracy especificamente.
- **`anchor test` tenta validator local** — Bankrun não precisa. Fix: rodar `npx ts-mocha` direto via `npm test`; `anchor test` reservado para `npm run test:full` quando integração com validator real for necessária.

## Caveats / dívidas técnicas

- **`#[allow(ambiguous_glob_reexports)]`** em `instructions/mod.rs` é dívida controlada. Quando M4+ adicionar mais instructions com `handler`, considerar renomear para `run` ou inline-call em `lib.rs`.
- **anchor-bankrun unmaintained** — versão 0.5 só suporta Anchor 0.30 oficialmente. Se 0.32 IDL parser quebrar para alguma instrução complexa em M4+, plano B é escrever um `BankrunProvider` minimal em ~50 linhas usando solana-bankrun direto.
- **Genesis flow ainda não testado end-to-end** — virá em M4 quando `mint` existir.
- **Sem teste para `set_valor` com `metadata` muito longo** (>200 bytes) — adicionar quando M4 começar a setar valors com strings reais. O check `require!(metadata.len() <= MAX_METADATA_LEN)` está implementado mas não exercido.
